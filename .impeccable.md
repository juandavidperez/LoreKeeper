# LoreKeeper — Design Context

## Design Context

### Users
Juan David, lector habitual de manga y novelas ligeras (actualmente en japonés/español). Usa LoreKeeper como diario de lectura personal: registra progreso semanal, anota personajes, lugares, citas, y reglas del mundo narrativo. El contexto de uso es íntimo — abre el app mientras lee o justo después de terminar un capítulo. Aunque la prioridad es el uso personal, las decisiones de diseño deben escalar para que otro lector pueda entrar sin contexto.

**Job to be done**: Capturar el conocimiento del mundo narrativo antes de que se olvide. La app es un grimorio vivo que crece con cada sesión de lectura.

### Brand Personality
**Solemne · Vivo · Íntimo**

La voz de LoreKeeper es la de un archivero anciano que toma muy en serio su oficio — pero que también se emociona cuando descubre una conexión entre personajes. No es frío ni burocrático; tampoco es juguetón ni casual. Es una herramienta que trata el acto de leer como algo que merece rituales.

El vocabulario canónico: Crónicas (entradas), El Archivo (enciclopedia), Oráculo (IA), SELLADO (completado), Crónicas de viaje, Forjar, Invocar, Desvanecer, el Éter (conexión de red).

### Aesthetic Direction

**Canónico**: Grimorio dorado sobre fondo oscuro / pergamino medieval
- Dark mode: `#0c0a08` base, `#f59e0b` amber como acento sagrado
- Light mode: `#f4ead5` pergamino, `#b45309` amber oscuro — **esta es la paleta más distintiva y debe elevarse**
- Tipografía: Playfair Display para títulos (editorial, literario), Inter para UI, Source Serif 4 para cuerpo de lectura

**El modelo estético**: La página offline `Archivo Silencioso` está más cerca del ideal que el app principal. El doble borde de manuscrito, el fondo pergamino con textura, el copy solemne — eso es la dirección. El app principal hereda demasiado del scaffold genérico de Tailwind.

**Referencias positivas**:
- Manuscritos iluminados medievales (ornamentación con propósito, jerarquía visual por tamaño físico)
- The Night Circus (Erin Morgenstern) — prosa densa, atmósfera que envuelve
- Editorial de libros académicos de historia — seriedad sin frialdad
- Notion si hubiera sido diseñado para un archivero de fantasía

**Anti-referencias (todos confirmados)**:
- ❌ Apps de productividad genéricas (Notion, Obsidian plain, Todoist) — funcionales sin alma
- ❌ Dashboards corporativos — métricas frías, gráficas de barra, tablas de reporte
- ❌ Estética anime/gaming — neon sobre negro, efectos glow, HUD de videojuego
- ❌ Social media / feed — cards infinitas, likes, notificaciones, espacio público

**Nota de inconsistencia**: El favicon usa violet `#863bff` que no pertenece al sistema de color canónico amber/pergamino. Debe alinearse con la identidad dorada o reemplazarse.

### Design Principles

1. **El amber es sagrado, no decorativo.**
   Amber es el único acento, lo que significa que cada uso compite por atención. Actualmente se usa para 10+ propósitos distintos. Debe reservarse para: estado activo, la acción primaria y momentos de logro. Todo lo demás debe comunicarse con variaciones de tono dentro del sistema zinc/parchment.

2. **La UI es parte del mundo narrativo.**
   El usuario está inmerso en una historia. La interfaz no debe romper esa inmersión con patrones de productividad genéricos. Cada string de UI, cada gesto de interacción, cada estado vacío debe sentirse coherente con el vocabulario del grimorio. La experiencia de *usar* el app es una extensión de la experiencia de *leer*.

3. **El contenido primero, el chrome mínimo.**
   En mobile (el contexto principal de uso), el chrome — headers, navbars, sticky panels — reduce el espacio para lo que importa: las crónicas, los personajes, la semana actual. Reducir capas de navegación es una prioridad funcional y estética.

4. **El pergamino lidera, el oscuro sigue.**
   El modo pergamino (`#f4ead5`) es la paleta más auténtica y distintiva de LoreKeeper. No debe ser un "modo alternativo" — debe ser la experiencia de referencia desde la que se evalúa todo el diseño. El dark mode es válido pero no debe dictar las decisiones de sistema.

5. **Los momentos de deleite son rituales, no confeti.**
   Las animaciones, los estados de éxito y los micro-interactions deben tener la solemnidad de un ritual, no la urgencia de una notificación push. Una semana sellada merece una pausa, no un bounce. El Oráculo respondiendo merece transición, no un spinner genérico.
